# 角色塔防·自走棋羁绊系统设计文档

## 1. 概述
本项目将传统塔防的“塔”替换为可收集、养成、拥有标签与技能的角色（英雄），并融合自走棋的羁绊系统。玩家通过招募英雄、凑齐羁绊、升星强化，构建一支独特的防御阵容，抵御一波波敌人。游戏旨在提供塔防的策略布局与自走棋的收集养成双重乐趣。

## 2. 核心玩法
- 玩家在战场上的固定格子放置英雄，英雄自动攻击沿固定路线前进的敌人。
- 英雄拥有多个标签（如“人类”、“法师”、“野兽”等），场上英雄的标签组合满足条件时激活羁绊效果（全局或区域增益）。
- 每波敌人结束后进入准备阶段，玩家可用金币从商店购买英雄，并管理阵容（放置、移动、出售、升星）。
- 通过合理搭配英雄标签、升级星级、激活强力羁绊，防守住所有波次。

## 3. 游戏流程
1. **准备阶段**（波次之间）
   - 商店自动刷新，显示一组随机英雄。
   - 玩家购买英雄，暂存于待部署区。
   - 从待部署区拖拽英雄到战场格子，或调整已有英雄位置。
   - 可出售英雄回收金币。
   - 可花费金币手动刷新商店，或锁定商店防止自动刷新。
   - 当拥有三个相同英雄（同星级）时，可合成升星，提升属性和能力。
   - 点击“开始战斗”进入战斗阶段。
2. **战斗阶段**
   - 敌人按波次生成，沿路线前进。
   - 英雄自动攻击射程内的敌人。
   - 战斗结束后根据杀敌数、漏敌数结算金币、经验（可选）和装备掉落。
   - 进入下一波准备阶段。

## 4. 英雄（角色）系统

### 4.1 英雄属性
- **基础属性**：攻击力、攻击速度、攻击范围、生命值（可能无生命，塔防风格）、移动速度（通常为0，固定位置）。
- **标签列表**：用于羁绊判定，例如“人类”、“精灵”、“战士”、“法师”等。
- **技能**：每个英雄拥有一个主动或被动技能（如范围伤害、减速、吸血等）。
- **星级**：初始1星，通过合成升星，提升基础属性、技能效果，可能增加新标签。
- **等级**（可选）：英雄通过杀敌获得经验升级，每次升级提升少量属性或解锁天赋。

### 4.2 英雄数据资源
- 使用 Godot 的 `Resource` 类型定义 `HeroData`，包含：
  - 名称、图标、模型/动画资源
  - 基础属性（范围值，或固定值）
  - 初始标签列表
  - 技能定义（技能ID、参数、描述）
  - 升星所需同名英雄数量（通常为3）
  - 升星后新增标签（可选）
  - 稀有度（影响商店出现概率和价格）

### 4.3 英雄实例
- 每个英雄实例是一个 `Node2D` 节点，持有 `HeroData` 引用，并包含：
  - `BuffComponent` 管理临时效果（羁绊、装备等）。
  - `AnimationPlayer` 控制攻击、施法、受击等动画。
  - `Area2D` 用于攻击范围检测。
  - 动态属性（当前生命、星级、等级等）。

## 5. 羁绊系统

### 5.1 羁绊定义
- 每个羁绊由若干条件（标签及数量）和对应的效果组成，可有多级（如2/4/6人类分别激活不同效果）。
- 效果可以是：
  - 数值加成：攻击力%、攻速%、射程%、生命值%等。
  - 技能增强：冷却缩减、伤害增加、额外效果。
  - 特殊机制：吸血、护甲穿透、暴击、光环等。
  - 组合技：特定条件触发全屏技能或召唤物。

### 5.2 羁绊管理
- `SynergyManager` 全局单例：
  - 维护当前场上所有英雄的列表及其标签。
  - 监听英雄添加/移除/升星事件，重新计算所有羁绊激活状态。
  - 维护激活的羁绊列表及效果对象。
  - 向受影响的英雄发送应用/移除效果的消息。
- 英雄内部的 `BuffComponent` 负责管理收到的效果，并修改最终属性或攻击行为。

### 5.3 羁绊UI
- 右侧面板显示当前激活的羁绊名称、等级、效果描述。
- 鼠标悬停英雄时，显示该英雄参与激活的羁绊图标。
- 商店中英雄卡片悬停时，显示该英雄可激活的羁绊（基于当前阵容）。

### 6.2 商店交互
- **刷新**：每波结束后自动刷新（未锁定时）。玩家可花费金币手动刷新，刷新后商店英雄重置。
- **锁定**：点击锁定按钮，下一波结束后商店不会自动刷新，直到解锁或手动刷新。
- **英雄卡片**：显示头像、名称、标签图标、价格。悬停显示详细属性及羁绊关联。
- **购买**：点击卡片，若金币足够且待部署区未满，则扣除金币，英雄添加到待部署区。若金币不足或待部署区满，给出提示。

### 6.3 待部署区交互
- 容量限制（如8格），显示已购买但未放置的英雄。
- **放置**：拖拽英雄到战场上的空格子，英雄从待部署区移除，战场生成英雄实例。
- **移动**：已放置的英雄可拖拽到其他空格子（交换位置）。
- **出售**：右键点击英雄（战场或待部署区），确认后获得一半价格金币，英雄消失。

### 6.4 升星合成
- **条件**：场上和待部署区中，相同名称且同星级的英雄数量达到3个。
- **提示**：可合成的英雄高亮（金色边框+脉冲），屏幕出现合成按钮（或悬停显示合成预览）。
- **预览**：悬停合成按钮或高亮英雄时，显示合成后英雄的星级、属性变化、新增标签、技能强化描述。
- **执行**：点击合成按钮，消耗3个英雄，在待部署区生成一个升星后的英雄（若待部署区满则自动放置到战场空位，若全满则取消并提示）。播放合成特效，更新羁绊。

### 6.5 交互反馈
- 购买/出售/合成播放相应动画和音效。
- 金币变化时数字跳动。
- 英雄状态变化（星级提升）更新头像和模型效果。
- 羁绊激活/失效时，羁绊面板刷新并播放提示。

## 7. 技术架构概览

| 模块 | 职责 | 关键类/节点 |
|------|------|-------------|
| 游戏主循环 | 管理波次、准备/战斗状态切换 | `GameManager` |
| 战场管理 | 管理英雄放置、路径、敌人生成 | `Battlefield`, `PathFollow2D` |
| 英雄管理 | 英雄实例创建、销毁、属性修改 | `Hero`, `HeroData`, `BuffComponent` |
| 羁绊管理 | 统计标签、激活羁绊、分发效果 | `SynergyManager` (全局单例) |
| 商店系统 | 管理英雄池、刷新、购买 | `ShopUI`, `ShopManager` |
| UI系统 | 显示羁绊、英雄属性、波次、金币等 | 各种UI节点 |

## 8. 实现路线图

### Phase 1：基础框架
- [ ] 实现固定路线敌人移动和基础攻击。
- [ ] 创建3-5种英雄，固定位置放置。
- [ ] 实现简单的羁绊（单一标签计数），效果为攻击力加成。
- [ ] 实现基础UI（金币、波次、羁绊显示）。
- [ ] 实现英雄购买与放置的基础交互。

### Phase 2：角色与羁绊扩展
- [ ] 为英雄添加技能系统（先实现被动技能）。
- [ ] 扩展羁绊条件（多标签组合、多级效果）。
- [ ] 实现完整的商店系统（刷新、锁定、待部署区）。
- [ ] 实现升星系统（合成检测、UI提示、属性提升）。

### Phase 3：养成与表现
- [ ] 增加英雄等级与经验（可选）。
- [ ] 增加装备系统（基础属性加成）。
- [ ] 丰富英雄动画与特效。
- [ ] 增加更多英雄与羁绊。

### Phase 4：优化与平衡
- [ ] 性能优化（对象池、减少每帧操作）。
- [ ] 数值平衡调整，设计Boss波次。
- [ ] 添加新手引导和音效。
- [ ] 多平台适配（如移动端触摸操作）。

## 9. 设计注意事项
- 英雄比塔更复杂，需确保每个英雄实例的性能开销可控。
- 羁绊效果应用要清晰，避免多个buff叠加导致数值爆炸。
- UI需直观展示英雄的羁绊参与情况，降低玩家认知负担。
- 升星和装备系统会增加策略深度，需有适当的引导和反馈。

## 10. 总结
本设计将塔防的固定防御与自走棋的羁绊养成相结合，以英雄为核心载体，通过商店购买、升星强化、羁绊搭配，创造出丰富的策略空间。技术实现上，基于 Godot C# 构建模块化系统，分阶段迭代，先验证核心玩法，再逐步扩展内容。该设计具备可扩展性，未来可加入更多英雄、羁绊、装备、特殊事件等，形成独特的游戏体验。
